# Postmortem 0001 — the map's "Current state" named an already-landed scope item as next work

- Date: 2026-07-05
- Stage: 2 — Growth
- Failure: [AGENTS.md](../../AGENTS.md) is the map — a fresh agent's daily entry point (LAW-4), and its "Current state" section states the *correct first action*. While building [plan 0003](../plans/completed/0003-growth.md) scope item 3, that section still read "the next work is scope item 2, **repo-fitness**" — but scope item 2 had already landed (commit 482a4d6) and the plan's own Next actions marked it ✅. The map's hand-authored current-state prose had drifted from the plan's actual progress: a fresh agent entering through the map would have been pointed at completed work. It was not caught by any check — the doc-gardener drift instrument ([`.seed/checks/doc-drift.ts`](../../.seed/checks/doc-drift.ts)) is v0 and detects stale *path* references, not a stale prose claim about which step is next.
- Fix: Corrected the map's "Current state" to reflect scope items 1–3 (planted) and name scope item 4 as next — landed under [plan 0003](../plans/completed/0003-growth.md) as part of the scope-item-3 commit (123738a). The immediate misdirection is gone; the *class* — hand-maintained current-state prose drifting from plan progress — is addressed by the invariant.
- Invariant: This is a recurrence of the "broader prose drift" divergence shape already priced as [E-009](../plans/entropy-ledger.md), which the v0 drift instrument does not yet detect. Prevention is **doc-only** for now: the map's current-state is deliberately hand-authored orientation (not a derived artifact), so the discipline is *update it in the same commit that lands a scope item*, and the recurrence is E-009's own stated trigger to grow a prose-state drift class when the shape proves to recur. — Enforcement: doc-only — justified: prose-state agreement is not cheaply mechanizable and the drift class does not exist yet; the residual is priced as [E-009](../plans/entropy-ledger.md) with a concrete conversion path, so an unenforced rule does not masquerade as enforced (LAW-2). The mechanizable *sibling* — the growth **stage** stated in two unchecked places — has its own cheap path as [E-011](../plans/entropy-ledger.md).
- Ring: The decision to handle this doc-only, why not yet mechanical, and the conversion trigger — [ring 0018](../rings/0018-map-current-state-drift-doc-only.md).
